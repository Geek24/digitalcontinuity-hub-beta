# Hub Beta · Compliance & Privacy Audit

**Date:** 2026-04-25
**Scope:** `hub-beta.digitalcontinuity.ai` (this repo, Next.js rebuild on `rebuild/nextjs-stack`)
**Reviewer:** Manual review only. **No third-party WCAG engine was run** against this beta and no automated WCAG/axe-core claims are made here.

## Summary

This is a prototype-grade beta. It contains no live API calls, no auth, no production data, no tracking, and no analytics. The full surface is fixture-driven and served from a Vercel preview / production deployment with a `noindex, nofollow, noarchive` directive on every route.

## Tech-stack parity with production

The beta is built on the same stack as the SmarterTariff production app:

- Next.js ^16.2.4 (App Router)
- React 19.2.5
- TypeScript ^5.9.3
- Tailwind CSS ^4 (CSS-first config)
- ESLint ^9 + `eslint-config-next@16.2.4`
- Utility libs: `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react`

This is intentional. By matching prod exactly, the beta cannot drift the dependency surface that the production security/IP scanner already vets. No Vite, no React Router, no axe-core, no Playwright, no testing framework wrappers — none of those exist in production, so they don't exist here.

## GDPR / privacy posture

| Area | Status | Notes |
|---|---|---|
| Cookies | None set | No analytics, no ads, no consent banner needed |
| Tracking pixels | None | No GA, no PostHog, no Vercel Analytics, no Sentry |
| Forms collecting PII | None | The "scan URL" input on Landing is a static demo string |

| `localStorage` | One key, `dc-hub-beta:tweaks` | Stores last chosen theme/mode/density/etc. so reloads remember preferences. Cleared by clearing site data. No PII. |

GDPR Article 4 "personal data" is not collected, processed, or stored. There is no controller / processor relationship to declare for this beta. Production has its own DPA on `smartertariff.ai`.

## Indexing & crawling

Every route is gated by:

1. `metadata.robots.{ index: false, follow: false, nocache: true, googleBot: { index: false, follow: false, noimageindex: true } }` (in `app/layout.tsx`) — emits `<meta name="robots" content="noindex,nofollow,nocache">` and a `googlebot` variant
2. `X-Robots-Tag: noindex, nofollow, noarchive` HTTP header (in `vercel.json`)
3. `public/robots.txt` with `Disallow: /`

This redundancy is deliberate: the meta tag, the header, and the robots file each defeat a different class of crawler.

## Security headers

`vercel.json` sets the following on every response:

- `X-Robots-Tag: noindex, nofollow, noarchive`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: SAMEORIGIN`
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`

No Content-Security-Policy is set on the beta. CSP is a production concern; tightening it on the beta has historically broken the Vercel preview tooling and the static prototype has no payment flow, no auth, and no third-party iframes that would need to be allow-listed.

## Accessibility — manual review only

A manual review was conducted against the WCAG 2.2 AA checklist on Landing, Platform, and Hub at the four palettes × light/dark combinations. The results below are the reviewer's observations, not an automated tool's output.

| Item | Status | Note |
|---|---|---|
| Skip-to-main link | Present | First focusable element on every page (`<a class="skip-link" href="#main">`) |
| Heading order | OK | Each page has one `<h1>`; `<h2>` sections labelled by `aria-labelledby` |
| Focus visibility | Tuned | Global `:focus-visible` outline at 2px solid `var(--accent)`, offset 2px |
| Reduced motion | Honoured | `@media (prefers-reduced-motion: reduce)` zeros transitions / animations; the Hub scan-simulation jumps straight to the done state |
| Keyboard nav | OK | Tool sidebar in the Hub responds to ↑/↓; megamenu on Platform closes on Escape; Variants drawer closes on Escape and on overlay click |
| Color contrast | Tuned in tokens | Contrast ratios in `app/styles/tokens.css` were tuned during the v0.1 design pass; they have **not** been re-verified by an automated tool in this beta |
| Names + roles | OK | Buttons, regions, dialogs, meters, progressbars all have explicit ARIA |

If a third-party accessibility audit is required for compliance evidence, run it against the live production site at `smartertariff.ai`, not against this beta.

## Best practices

- **No `dangerouslySetInnerHTML`** anywhere in the codebase
- **No client-side fetches** — the entire app is renderable without network
- **No sensitive data in fixtures** — `lib/fixtures.ts` only contains values that are visible on `smartertariff.ai` today
- **Patent-pending IP guardrails** — priority titles strip internal numeric identifiers and surface only the public-facing WCAG criterion. Scoring weights, signal-detection logic, and criterion → revenue mappings are not exposed in this beta.

## Known gaps

- Browser-level keyboard reachability has not been tested under JAWS/NVDA on this beta. The atoms come from the v0.1 design pass which was reviewed manually; production carries its own JAWS/NVDA evidence.
- Color contrast tokens are inherited from v0.1 unchanged. Re-running an automated contrast pass is recommended before any external audit references this beta.
