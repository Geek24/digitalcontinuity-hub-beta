## B. GDPR + privacy audit

### Methodology
- Inspected response headers and HTML of every public route via `curl`
- Grepped built JS bundle for third-party origins
- Inspected `localStorage` / `sessionStorage` writes in source

### Cookies
**None set by the application.** Vercel may set a single `_vercel_jwt` cookie for password-protected previews (not on the public beta). Verified: `Set-Cookie` header absent on `GET /`, `GET /app`, `GET /platform`.

### Third-party network requests
**Zero.** A grep of `dist/assets/index-*.js` for `https?://` returns only:
- `http://www.w3.org/...` (XML namespaces, used by React DOM internals — not network calls)
- `https://hub-beta.digitalcontinuity.ai/...` (constructed for the in-app share-this-cut mailto link)
- `https://reactjs.org/docs/error-decoder.html` (only used if React throws a production-mode error; never auto-fetched)

No analytics, no Sentry, no Hotjar/Mixpanel/Segment, no font CDN, no Stripe, no Google Tag Manager.

### Storage writes
- `localStorage["dc-hub-beta:tweaks"]` — JSON of the user's preferred theme/mode/hero/etc. **No PII.** Used to remember the variant cut between visits. Acceptable per beta hygiene; documented in `src/lib/tweaks.ts`.
- `sessionStorage` — not used.

### Fonts
System-font stack via CSS tokens (`var(--font-sans)`, `var(--font-serif)`, `var(--font-mono)`). No `@font-face` rules and no requests to `fonts.googleapis.com` / `fonts.gstatic.com`.

### Tracking pixels in `<head>`
**Zero.** The `<head>` contains: `<meta charset>`, `<meta viewport>`, `<meta name="robots">`, `<meta name="x-environment">`, `<meta name="description">`, `<title>`, the bundled `<script>`, and the bundled `<link rel="stylesheet">`. Nothing else.

### PII
No forms accept input. The only outbound user-initiated traffic is the `mailto:` link in the beta banner, which uses the system mail client and never touches the app.

### Cookie banner / privacy policy
**Not required for this beta** (no PII collected, no tracking, no non-essential cookies). A footer link to a generic `/privacy` page is a v1.0 concern.

### Verdict
**No GDPR concerns.** The site is functionally a static brochure with client-side variant prefs.

## C. Best practices audit

### Security headers (live deploy, before v0.2)

```
HTTP/2 200
strict-transport-security: max-age=63072000   ← good (2 years), but no includeSubDomains/preload
x-robots-tag: noindex, nofollow                ← good
```

**Missing on live:**
- `Content-Security-Policy` — no CSP at all
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options` (or CSP `frame-ancestors`)
- `Referrer-Policy`
- `Permissions-Policy`

**Fixed in this PR (`vercel.json`):**
- CSP: `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self' data:; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'`
  - `'unsafe-inline'` retained for styles only (React inline `style` attributes from the artboard port — would require nonce strategy to remove; tracked for v1.0).
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY` (defence in depth alongside CSP `frame-ancestors 'none'`)
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=(), usb=(), browsing-topics=()`
- `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload` (HSTS-preload eligible)

### HTTPS-only
✅ HTTP→HTTPS forced by Vercel. HSTS now preload-eligible after v0.2 deploys.

### Bundle size
- v0.1 (live): ~64 KB gzip JS, ~3.6 KB gzip CSS
- v0.2 (this PR): 71.21 KB gzip JS, 4.74 KB gzip CSS
- Ceiling: 200 KB gzip
- ✅ **64% headroom remaining.**

### Lighthouse-equivalent checks
- ✅ Image alt text: no `<img>` tags in the build (all icons are CSS/SVG with `aria-hidden` or `aria-label`).
- ✅ Link discernible names: all `<a>` and `<button>` either have visible text or `aria-label`.
- ✅ Form labels: no `<input>` elements (display-only mocks).
- ✅ `<html lang="en">`.
- ✅ `<title>` present and descriptive.
- ✅ `meta description` present.

### `robots.txt`
- v0.1 (live): `GET /robots.txt` returned the SPA HTML (the catch-all rewrite swallowed it). Search engines were still blocked via the `noindex` meta + `X-Robots-Tag` header, but `robots.txt` was technically broken.
- v0.2 (this PR): `public/robots.txt` added with `User-agent: * / Disallow: /`, and `vercel.json` rewrites updated to serve it before the catch-all.

### `humans.txt`
✅ Present at `/humans.txt`, served correctly.

## Suggested follow-ups (not in v0.2 scope)

1. **CSP `style-src` without `'unsafe-inline'`** — requires a nonce strategy or migration of inline `style={...}` to CSS classes. Defer to the v1.0 production port.
2. **VoiceOver/NVDA recordings** for the Variants drawer + Hub tool switcher. Manual; file with the design team.
3. **Sub-resource Integrity (SRI)** on the bundled `<script>` and `<link>` — Vite supports it via plugin; small win for a static beta.
4. **Privacy-policy stub** at `/privacy` even though no PII is collected — preempts trust questions when the beta is shared with prospects.
